# Money Keeper Clone - Supabase API Plan

## Overview

This document outlines the API design for the Money Keeper Clone app. All APIs use Supabase's auto-generated REST API with custom PostgreSQL functions for complex operations.

---

## 1. Database Schema

### Tables
- `profiles` - User profile (extends auth.users)
- `wallets` - Wallet accounts
- `categories` - Transaction categories (system + custom)
- `transactions` - Income/expense records
- `budgets` - Budget limits
- `savings_goals` - Savings targets

### Row Level Security (RLS)
All tables have RLS enabled. Users can only access their own data.

---

## 2. REST API (Auto-generated by Supabase)

### Profiles
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/profiles?select=*` | Get current user profile |
| PATCH | `/profiles?select=*` | Update profile |

### Wallets
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/wallets?select=*,wallet_balance` | Get all wallets with balance |
| POST | `/wallets` | Create new wallet |
| PATCH | `/wallets?id=eq.{id}` | Update wallet |
| DELETE | `/wallets?id=eq.{id}` | Delete wallet |

### Categories
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/categories?select=*&is_system=eq.true` | Get system categories |
| GET | `/categories?select=*&user_id=eq.{user_id}` | Get user categories |
| POST | `/categories` | Create custom category |
| PATCH | `/categories?id=eq.{id}` | Update category |
| DELETE | `/categories?id=eq.{id}` | Delete category |

### Transactions
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/transactions?select=*,wallets(*),categories(*)` | Get all transactions |
| GET | `/transactions?user_id=eq.{user_id}&transaction_date=eq.{date}` | Filter by date |
| GET | `/transactions?user_id=eq.{user_id}&type=eq.{type}` | Filter by type |
| GET | `/transactions?user_id=eq.{user_id}&category_id=eq.{id}` | Filter by category |
| POST | `/transactions` | Create transaction |
| PATCH | `/transactions?id=eq.{id}` | Update transaction |
| DELETE | `/transactions?id=eq.{id}` | Delete transaction |

### Budgets
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/budgets?select=*,categories(*)` | Get all budgets |
| POST | `/budgets` | Create budget |
| PATCH | `/budgets?id=eq.{id}` | Update budget |
| DELETE | `/budgets?id=eq.{id}` | Delete budget |

### Savings Goals
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/savings_goals?select=*` | Get all savings goals |
| POST | `/savings_goals` | Create savings goal |
| PATCH | `/savings_goals?id=eq.{id}` | Update goal (add to current_amount) |
| DELETE | `/savings_goals?id=eq.{id}` | Delete goal |

## 3. PostgreSQL Functions (RPC)

### get_wallet_balance(wallet_id UUID)
Returns DECIMAL - Current balance of a wallet.

```sql
get_wallet_balance('uuid-here')
-- Returns: 3500000.00
```

### get_monthly_report(user_id UUID, month TEXT)
Returns JSON - Monthly report with income, expense, and category breakdown.

```sql
get_monthly_report('user-uuid', '2025-04')
-- Returns:
{
  "month": "2025-04",
  "total_income": 25000000.00,
  "total_expense": 15000000.00,
  "net_balance": 10000000.00,
  "by_category": [...]
}
```

### get_wallet_transactions(wallet_id UUID, start_date DATE, end_date DATE)
Returns SETOF transactions - All transactions for a wallet within date range.

```sql
get_wallet_transactions('wallet-uuid', '2025-04-01', '2025-04-30')
```

### get_category_stats(user_id UUID, start_date DATE, end_date DATE)
Returns JSON - Spending stats by category.

```sql
get_category_stats('user-uuid', '2025-04-01', '2025-04-30')
-- Returns: [{ "category": "Food", "total": 5000000, "count": 25 }, ...]
```

### update_savings_progress(goal_id UUID, amount DECIMAL)
Returns savings_goals - Add amount to current savings goal.

```sql
update_savings_progress('goal-uuid', 1000000)
```

### get_budget_status(user_id UUID, budget_id UUID)
Returns JSON - Budget usage percentage and remaining amount.

```sql
get_budget_status('user-uuid', 'budget-uuid')
-- Returns: { "budget_amount": 5000000, "spent": 3200000, "remaining": 1800000, "percentage": 64 }
```

## 4. API Usage Examples

### Create Transaction
```typescript
const { data, error } = await supabase
  .from('transactions')
  .insert({
    user_id: user.id,
    wallet_id: 'wallet-uuid',
    category_id: 'category-uuid',
    type: 'expense',
    amount: 150000,
    description: 'Lunch',
    transaction_date: '2025-04-29'
  })
  .select()
  .single()
```

### Get Wallet with Balance
```typescript
const { data: wallets } = await supabase
  .from('wallets')
  .select('*, wallet_balance: get_wallet_balance(id)')
```

### Get Monthly Report
```typescript
const { data } = await supabase
  .rpc('get_monthly_report', { 
    p_user_id: user.id, 
    p_month: '2025-04' 
  })
```

### Filter Transactions by Date Range
```typescript
const { data } = await supabase
  .from('transactions')
  .select('*, wallets(*), categories(*)')
  .eq('user_id', user.id)
  .gte('transaction_date', '2025-04-01')
  .lte('transaction_date', '2025-04-30')
  .order('transaction_date', { ascending: false })
```

## 7. Notes

- All monetary values stored as DECIMAL(15,2) - up to 999,999,999,999.99
- UUID primary keys for all tables
- Timestamps use TIMESTAMPTZ for timezone awareness
- RLS policies ensure data isolation between users
- Supabase auto-generates CRUD APIs - no manual REST needed

## 8. Deploy Schema to Supabase

### Option A: GitHub Actions (Auto-deploy on push)

1. **Add Supabase CLI to GitHub Actions**:
   ```yaml
   # .github/workflows/deploy-supabase.yml
   name: Deploy Supabase Schema
   on:
     push:
       branches: [main]
       paths:
         - 'supabase/**'
   jobs:
     deploy:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: supabase/setup-cli@v1
           with:
             supabase-version: latest
         - run: |
             supabase link --project-ref ${{ secrets.PROJECT_REF }}
             supabase db push
           env:
             SUPABASE_ACCESS_TOKEN: ${{ secrets.SUPABASE_ACCESS_TOKEN }}
   ```

2. **Add GitHub Secrets**:
   - `PROJECT_REF` - Found in Supabase Dashboard → Settings → General
   - `SUPABASE_ACCESS_TOKEN` - Create in Supabase Dashboard → Settings → Access Tokens

3. **Migration files** go in `supabase/migrations/` with timestamp prefix (e.g., `20250429000000_initial_schema.sql`)

### Option B: Manual (No CI/CD)

1. Keep `docs/supabase-schema.sql` as single file
2. Run manually in Supabase Dashboard → SQL Editor
3. Simple but requires manual steps

### Migration File Format

```sql
-- supabase/migrations/20250429000000_initial_schema.sql

-- Enable UUID extension
CREATE EXTENSION IF NOT EXISTS "pgcrypto";

-- Profiles, Wallets, Categories, Transactions, Budgets, Savings Goals...
-- (Your full schema here)
```

Each push to `main` with changes in `supabase/` folder triggers the workflow.
